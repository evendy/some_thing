解决git中文乱码问题 进入git安装目录，改一下配置就可以基本解决：

1、etc\gitconfig：

[gui] encoding = utf-8 [i18n] commitencoding = utf-8 [svn] pathnameencoding = utf-8 说明：打开 Git 环境中的中文支持。pathnameencoding设置了文件路径的中文支持。

2、etc\git-completion.bash：

alias ls='ls --show-control-chars --color=auto' 说明：使得在 Git Bash 中输入 ls 命令，可以正常显示中文文件名。

3、etc\inputrc：

set output-meta on set convert-meta off 说明：使得在 Git Bash 中可以正常输入中文，比如中文的 commit log。

4、etc\profile：

export LESSCHARSET=utf-8

配置本地git信息

git config --global user.name "yejianjie" //配置用户名 git config --global user.email "yejianjie@appvv.com" //配置用户邮箱

git config --global core.autocrlf false //禁用自动转换(windows中的换行符为 CRLF， 而在linux下的换行符为LF)

git基础命令 首先配置本地git的信息

下载playhub代码 git clone git@gitlab.appvv.com:sz-android-development/playhub.git

查看本地和远程分支（*为当前所处分支） git branch -a

新建分支 git branch “branchname”

切换分支 git checkout “branchname”

基于当前分支创建一个新的分支并切换到该分支 git checkout -b “branchname”

查看本地代码与git库代码的差异文件列表 git status

查看git库的历史提交记录 git log

查看git库的详细历史提交记录 gitk

提交文件到git库 git add “filename”

提交当前全部差异文件到git库(慎用) git add .

确认当前的代码提交 git commit -m “commit message”

提交代码到远程分支 git push origin “branchname”

拉取git库最新的代码 git pull origin “branchname”

回滚git版本到某个log节点 git reset --hard “commit hashcode”

列出所有的tag git tag

给git版本打tag git tag “tag_name”

将本地tag推送到git库 git push origin “tag name”
